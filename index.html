🐠 Goldfish G+ (Buildup)

Memoryless Markov transition model · WC 2026

⚠️ Proof of concept — Buildup-only model. Insights are based on limited training data (3–6 matches per team). Not predictive. For methodology demonstration only.

Data provided by StatsBomb Open Data.

How it works ▾

Every on-ball action in a match (pass, carry, duel, etc.) is a state in a Markov chain, defined by action type, pitch zone, and which team has the ball. We compute absorption probabilities: given possession is in state X, what's the probability it ends in a goal for vs. against?

A hierarchical Bayesian prior (2.2M transitions from 314 matches across World Cup 2018 & 2022, Euro 2020 & 2024, AFCON 2023, and Copa América 2024) regularizes each team's observed transitions. Team-specific updates use World Cup 2022 plus each confederation's most recent continental tournament (Euro 2024, AFCON 2023, Copa 2024).

The advantage is how much a team's net scoring probability from a state exceeds the prior average. Positive = strength, negative = weakness.

Shots that directly led to goals are excluded — the insight is in the buildup patterns that shift scoring probability before the final shot.

Buildup-only model: This version goes further — goal-scoring and goal-conceding transitions are excluded entirely from each team's observed data when updating the posterior. The prior still contains all transitions (including goals), so absorption probabilities resolve correctly, but teams are differentiated only by their mid-chain buildup patterns: how they progress possession through passing, carrying, pressing, etc. This prevents the model from simply reflecting that a team scored a lot in a small sample, and instead surfaces the underlying tactical patterns that create (or prevent) scoring opportunities.

The GG+ is a summary score: each state's advantage weighted by how often an average team occupies that state (prior frequency). It represents expected net goal probability added per possession, scaled ×10,000 for readability. Positive = above average, negative = below.
